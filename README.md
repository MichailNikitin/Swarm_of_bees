# Симулятор роя пчёл

Интерактивный веб-симулятор мультиагентной системы. Пчёлы, цветы и улей взаимодействуют в реальном времени под управлением централизованного алгоритма Swarm Intelligence Controller.

---

## Быстрый старт

```
launcher.bat
```

Автоматически установит зависимости, запустит сервер на `http://localhost:8000` и откроет браузер.

**Требования:** Python 3.10+, доступ к интернету для первой установки пакетов.

## Бэкенд

### `backend/main.py`

FastAPI-приложение с тремя точками входа:

| Маршрут    | Тип       | Назначение                                  |
|------------|-----------|---------------------------------------------|
| `GET /`    | HTTP      | Отдаёт `frontend/index.html`                |
| `/static`  | HTTP      | Статические файлы фронтенда                 |
| `WS /ws`   | WebSocket | Двусторонний канал симуляции                |

WebSocket принимает команды от клиента (`start`, `stop`, `reset`, `update_params`) и рассылает снэпшоты состояния всем подключённым клиентам при каждом тике.

---

### `backend/simulation/agents.py`

Содержит датаклассы всех агентов системы:

#### `Bee` — пчела
| Поле             | Значение по умолчанию | Описание                            |
|------------------|-----------------------|-------------------------------------|
| `state`          | `idle`                | Текущее состояние автомата          |
| `nectar`         | `0.0`                 | Накопленный нектар (макс. 1.0 ед.)  |
| `max_nectar`     | `1.0`                 | Ёмкость                             |
| `collect_rate`   | `0.2`                 | Скорость сбора нектара (ед./тик)    |
| `unload_rate`    | `0.5`                 | Скорость разгрузки в улей (ед./тик) |

**Граф состояний:**
```
idle ──→ to_flower ──→ collecting ──→ to_hive ──→ unloading ──→ idle
```

#### `Flower` — цветок
| Поле          | Значение по умолчанию | Описание                          |
|---------------|-----------------------|-----------------------------------|
| `state`       | `open`                | Открыт / закрыт                   |
| `nectar`      | 2–5 ед. (случайно)    | Текущий запас нектара             |
| `max_nectar`  | `5.0`                 | Максимальный запас                |
| `regen_rate`  | `0.05`                | Скорость восполнения нектара      |

**Правила переключения состояния** (управляет контроллер):
- `nectar < 0.5` → закрыт
- `nectar > 3.0` → открыт

#### `Hive` — улей
Стационарный аккумулятор. Метод `process_nectar(amount)` добавляет нектар и конвертирует его в мёд по формуле **3 ед. нектара → 1 ед. мёда**.

---

### `backend/simulation/controller.py` — Swarm Intelligence Controller

Централизованный алгоритм управления. Вызывается один раз за тик **после** физического обновления агентов. Выполняет три шага:

1. **`_update_flower_states`** — переключает состояние цветов по порогам нектара.
2. **`_assign_idle_bees`** — назначает бездействующих пчёл (`idle`) к открытым цветам, приоритизируя цветы с **максимальным** запасом нектара.
3. **`_send_full_bees_to_hive`** — отправляет пчёл с полной ёмкостью нектара в улей.

Контроллер не хранит внутреннего состояния — каждый тик он читает полный снэпшот симуляции и выдаёт команды напрямую агентам.

---

### `backend/simulation/engine.py` — SimulationEngine

Асинхронный движок на `asyncio`. Основные ответственности:

| Метод                  | Описание                                                |
|------------------------|---------------------------------------------------------|
| `start()`              | Запускает asyncio Task с игровым циклом                 |
| `stop()`               | Отменяет Task, сохраняет состояние                      |
| `reset(params?)`       | Пересоздаёт всё состояние, опционально с новыми params  |
| `update_params(data)`  | Горячее обновление параметров без перезапуска           |
| `get_snapshot()`       | Сериализует текущее состояние в JSON-совместимый dict   |
| `_tick()`              | Один шаг: цветы → пчёлы → контроллер                   |

**Игровой цикл:**
```python
while running:
    t0 = time.monotonic()
    _tick()
    await broadcast(get_snapshot())
    await asyncio.sleep(max(0, 1/tick_rate - elapsed))
```

Частота тиков настраивается слайдером в диапазоне **1–30 тик/сек**. При максимальной нагрузке (30 пчёл + 20 цветов) обработка одного тика составляет < 1 мс.

## Фронтенд

### `frontend/js/canvas.js` — CanvasRenderer

Рендерер на `requestAnimationFrame` (цель — 60 FPS), полностью декуплирован от WebSocket. Принимает снэпшот через `setState(data)` и при следующем кадре отрисовывает его.

**Порядок отрисовки:**
1. Очистка холста
2. Пунктирные линии маршрутов (пчела → цель)
3. Цветы (лепестки + центр + полоса нектара)
4. Улей (шестиугольник + соты + счётчик мёда)
5. Пчёлы (круг + полосы + индикатор нектара)

### `frontend/js/websocket.js` — SwarmWS

Тонкая обёртка над нативным `WebSocket`. При обрыве соединения автоматически переподключается через 2 секунды. Предоставляет три callback-хука: `onOpen`, `onClose`, `onMessage`.

### `frontend/js/app.js`

Точка сборки фронтенда. Связывает WebSocket, Canvas и DOM:
- Синхронизирует слайдеры с параметрами сервера
- Отправляет размер холста на бэкенд при инициализации и ресайзе
- Обновляет список агентов в левой панели при каждом тике

## WebSocket API

Все сообщения — JSON. Клиент → сервер:

```jsonc
{ "action": "start" }
{ "action": "stop" }
{ "action": "reset", "params": { "num_bees": 15 } }
{ "action": "update_params", "params": { "bee_speed": 5.0 } }
```

Сервер → клиент (снэпшот, отправляется каждый тик):

```jsonc
{
  "tick": 142,
  "running": true,
  "bees":    [{ "id": "bee_0", "x": 312, "y": 210, "state": "collecting", "nectar": 0.6 }],
  "flowers": [{ "id": "flower_0", "x": 500, "y": 300, "state": "open", "nectar": 3.2, "max_nectar": 5.0 }],
  "hive":    { "x": 450, "y": 300, "nectar": 0.6, "honey": 4.0 },
  "stats":   { "total_nectar_collected": 38.4, "total_honey": 12.0, "active_bees": 8, "open_flowers": 4 },
  "params":  { "bee_speed": 3.0, "nectar_regen": 0.05, "num_bees": 10, "num_flowers": 5, "tick_rate": 10 }
}
```

## Расширение системы

**Новый тип агента:** добавьте датакласс в `agents.py`, зарегистрируйте в `engine.py` (`_init_state`, `get_snapshot`) и добавьте отрисовку в `canvas.js`.

**Новый алгоритм управления:** создайте класс с методом `tick(state: SimulationState)` и подставьте его в `engine.py` вместо `SwarmController`.

**Новый параметр:** добавьте поле в `SimParams`, обработку в `update_params()`, слайдер в `index.html` и запись в `sliders` в `app.js`.
